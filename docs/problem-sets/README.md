# Problem Sets

Problem sets are the missing practice layer between reading a question and surviving a real interview loop.

Questions test fragments. Problem sets test sequence, prioritization, trade-offs, debugging, and recovery under constraints.

---

## How To Use This Surface

```text
Read the brief -> timebox your solution -> write or sketch the answer -> compare against the rubric -> extend with follow-ups
```

- Use the question library when you want targeted drilling.
- Use module guides when you need concept depth and study order.
- Use problem sets when you want to simulate the work an interviewer expects you to reason through end to end.

## Practice Modes

| Mode | What it trains | Best for |
|---|---|---|
| Build Sprint | Design plus implementation sequencing | Applied rounds |
| Debugging Drill | Symptom isolation and repair | Incident and debugging rounds |
| System Design Board | Architecture, control plane, failure domains | Senior and architect rounds |
| Follow-up Loop | Defending trade-offs under pressure | Real panel behavior |
